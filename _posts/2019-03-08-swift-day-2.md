## My second day of Learning Swift - 8th March 2019

In my second Swift lecture I have learned about:
* more complex data types in Swift: Sets, Array, tuples and Dictionaries
* I have learned what distinguishes those and who to initialize them
* enums - These give a possibility to restraint a pool of possible values for a variable

```swift
import UIKit

// arrays

let john = "John Lennon"
let paul = "Paul McCartney"
let george = "George Harrison"
let ringo = "Ringo Starr"

let beatles: [String] = [john, paul, george, ringo]
beatles[1]

// type annotations are written in brackets

// sets
// differences to arrays; different: no order, no item can appear twice

let colors = Set(["red", "green", "blue"])

// It is not possible to add items twice
let colors2 =  Set(["red", "green", "blue", "red", "blue"])
// contains red, green, blue once

// tuples, you can't add or remove items, you can't change types, you can access itmes using numerical positions

let singer = (first: "Robbie", last: "Williams")

// you can access values by position (numerical) or by name
singer.0
singer.last

// arrays vs sets vs tuples
// tuples make sense if you want to store items of a fixed number and different type
var adress = (house: 29, street: "Natenestra√üe", city: "Rosche")
// if a collection of values that must be unique is required, use a set
var set = Set([1, 2, 3, 4, 5, 6])
// arrays are moste common, the order of the items matters and you can have duplicates in there
var array = ["Eric", "Erica", "Eric", "Terry", "Michaela"]

// dictionaries - store identifier, value - pairs
var heights: Dictionary = [
    "Lukas Gehrke": 1.89,
    "Jonas Gehrke": 1.86,
    "Burkhard Gehrke": 1.83,
    "Stefanie Gehrke": 1.80
]
// values are accessed by their identifier
heights["Jonas Gehrke"]

// default values in dictionarys
let favoriteIceCream = [
    "Paul": "Vanille",
    "Lea": "Chocolate"
]

favoriteIceCream["Paul"]
favoriteIceCream["Charlotte", default: "unknown"]

// Creating empty collections

// empty dictionary
var teams = [String: String]()
teams["Paul"] = "Red"

// empty integer
var results = [Int]()

// empty set
var words = Set<String>()
var numbers = Set<Int>()

// enumerations - Define groups of related values in a way that makes them easier to use
enum Result {
    case success
    case failure
}

var result4 = Result.failure

// Associated values attached to each case. Attach additional information to enums

enum Activity {
    case bored
    case running(destination: String)
    case talking(topic: String)
    case singing(volume: Int)
}

let hobby = Activity.singing(volume: 13)

// Enum Raw values
// The following enum stores integer values for each case automatically

enum Planet: Int {
    case mercury
    case mars
    case earth
    case venus
}

let earth = Planet(rawValue: 2)

// Now every item will be assigned a number starting from 1
// Otherwise we would start with 0

enum Planets: Int {
    case mercury = 1
    case mars
    case earth
    case venus
}
```
